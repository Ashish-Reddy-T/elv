## Key Design Decisions Made

### GATr (Phase 3 — decided, not yet implemented)
The GATr repo (`REPOS/geometric-algebra-transformer`) already implements the improved PGA
with join bilinear layers via `GeometricBilinear` in `gatr/layers/mlp/geometric_bilinears.py`.
This is the AISTATS 2024 fix that addresses basic PGA's expressivity limitation. No custom
implementation is needed — we wrap the existing `GATr` class directly.

Convergence mitigations for Phase 3:
1. `checkpoint=["block"]` — built-in gradient checkpointing in the GATr constructor
2. `join_reference="data"` — reference multivector from input centroid, more stable than `"canonical"` for indoor scenes
3. Staged unfreezing — GATr starts frozen in pre-alignment, only trains from SFT onwards
4. Input coordinate normalization — normalize 3D coords to unit ball before PGA embedding (Habitat world coords span metres, PGA is numerically happiest near unit scale)

PGA point embedding per token: e₀₁₂=x, e₀₁₃=y, e₀₂₃=z, e₁₂₃=1, all others=0 (homogeneous
point in projective geometric algebra).

Invariant extraction: scalar channels (32-dim) + MV norms (16-dim) = 48-dim per token → MLP
projector → 4096-dim.

Observed incompatibility:
- With cached einsum path planning enabled, GATr fails in this stack during

  This keeps model outputs valid while avoiding the failing shape-planner path.
